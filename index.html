МУССОН
Модульная система управления сетями связи онлайн

Подробнее

Система МУССОН предназначена для мониторинга сети,
в иерархии ITU TMN реализует зонтичную систему управления NMS+ с поддержкой функционала четырех уровней модели FCAPS

Функции системы

Аварии, неисправности и события
  • Непрерывный круглосуточный мониторинг состояния оборудования и услуг
  • Отображение аварии - со счётчиком количества срабатываний и группировками
  • Полнофункциональная фильтрация и поиск
  • Маскирование аварии на определённом оборудовании
  • Автоматический перезапрос сетевого элемента при наличии пропуска трапа
Конфигурация, настройки и состав оборудования
  • Состав оборудования - для анализа необходимого ЗИП, возможности перемещения
  • Список всех установленных лицензий - для анализа потребности расширения функционала и модернизации сетевого элемента
  • Cписки конфигурации оборудования, сгруппированные по функционалу
  • Сбор данных по расписанию, по запросу оператора или по событиям на сетевом элементе
  • Контроль достоверности собранных данных – алгоритмы проверки и дозагрузки информируют администратора системы о проблемах с загрузкой
Производительность и деградация работы оборудования
  • Статистический анализ загруженности и качества работы сети
  • Статистика суточных или интервальных значений параметров
  • Общий список загруженности полосы передачи данных % и Мб/с - на пролете, физическом порту и по VLAN
  • Создание отчета по перегрузке каналов - с количеством потерянных пакетов
  • Онлайн измерения уровней в реальном времени на проблемном участке
  • Качества связи между системой (в операторском облаке) и оборудованием региона
Управление безопасностью

Преимущества

Единая мультивендорная мультиобъектная системаа

Компоненты системы на основе кода российского происхождения или открытого кода

Возможен сбор данных с сетевого элемента, а не только с системы управления вендора

Интеграция оборудования на основе проприетарных API и протоколов

Определение проблемных мест на сети

Оптимизация ресурсов сети

Оптимизация расходов, поддержка оператора в автоматизации работы с данными

Спецификация системы

  • Поддержка SNMP(v1, v2c, v3), FTP, SSH, NetConf
  • Возможность сбора данных напрямую с NE
  • Загрузка пользовательских MIB (от вендора) NEC Pasolink, Ericsson Minilink, Huawei RNT
  • Загрузка структур данных, данных RMON (RFC2819)
  • Группировка и маскирование аварий
  • Автоматический перезапрос сетевого элемента из-за пропуска аварии
  • Поиск элементов по сегменту сети (Auto-Discovery)
  • Определение топологии сети, отображение на схеме сети
  • Геопривязка объектов на карте

Архитектура платформы

Платформа построена на базе современных систем разработки ПО с открытым кодом, таких как RabbitMQ, PostgreSQL, Timescale DB, Clickhouse.

В архитектуру заложена реализация отказоустойчивости, кластеризации, горизонтального и вертикального масштабирования. Состав функциональных блоков зависит от требований к системе и соответствия бизнес-логике работы оператора.
